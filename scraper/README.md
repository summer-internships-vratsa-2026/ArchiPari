# Автоматично обновяване на цените — как работи и какво остава да направиш
 
Сайтът чете цените от `data/products.json`. За да са "живи" и актуални, този
файл трябва редовно да се презаписва от скрипт, който взема цените от
сайтовете на магазините (Praktiker, Homemax, Inex, Gstroy,
Domestico, Bauhaus и др.) и ги записва в същия формат.
 
## Важно — прочети преди да пуснеш каквото и да е
 
1. **Не мога да върна готов, работещ „вземи цена от X“ скрипт за всеки
   магазин от тук.** В средата, в която работя в момента, нямам достъп до
   интернет, за да отворя реалните страници на тези 7+ магазина, да видя
   действителния HTML/JSON, който връщат, и да напиша точните селектори.
   Структурата на всеки сайт (класове, API-та, защити) се различава и се
   променя с времето — затова ти давам **работещ скелет**
   (`scrape_template.py`), който трябва да адаптираш за всеки магазин,
   след като разгледаш неговия сайт (или ми пратиш линк/HTML откъс, за да
   ти помогна да напиша точния селектор).
2. **Условия за ползване (ToS) и robots.txt.** Преди да пускаш автоматично
   събиране на данни от чужд сайт, провери неговите Общи условия и
   `https://домейн/robots.txt`. Някои вериги изрично забраняват
   автоматизирано събиране на цени. По-безопасни алтернативи:
   - официално API за партньори/afiliate, ако магазинът предлага такова;
   - фийд с продукти (Google Shopping XML/CSV), който много е-търговци
     публикуват официално;
   - директен контакт с магазина за партньорство за данни.
3. **Anti-bot защити.** Много сайтове ползват Cloudflare или подобни
   защити, които блокират прости `requests` заявки. Може да ти трябва
   `playwright`/`selenium` (реален браузър) вместо обикновено HTTP заявка.
   Пусни го отговорно — с паузи между заявките (`time.sleep`), не
   претоварвай сървъра на магазина.
## Структура
 
```
scraper/
  requirements.txt      -> Python пакети
  scrape_template.py     -> скелет за един магазин (адаптирай за всеки)
  run_all.py             -> пуска всички скрапери и записва data/products.json
```
 
## Стъпки за да „оживее“ автоматичното обновяване
 
1. Инсталирай зависимостите:
```bash
   pip install -r requirements.txt
```
2. За всеки магазин, направи копие на `scrape_template.py`
   (напр. `scrape_praktiker.py`) и:
   - отвори страницата на конкретния продукт/категория в браузър;
   - виж откъде идва цената (обикновен HTML таг, или AJAX/JSON заявка,
     която можеш да извикаш директно — обикновено е по-бързо и по-стабилно);
   - попълни `BASE_URL`, `fetch_page()` и `parse_price()` според реалната
     структура.
3. Регистрирай новия скрапер в `run_all.py`.
4. Пусни `python run_all.py` — той ще генерира нов `data/products.json`.
5. Насрочи го да се пуска автоматично, например с `cron` на Linux сървър:
```
   0 */6 * * * cd /път/до/сайта/scraper && /usr/bin/python3 run_all.py >> log.txt 2>&1
```
   (пример: на всеки 6 часа)
6. Ако сайтът се хоства статично (без бекенд), качвай обновения
   `data/products.json` при всяко пускане на cron задачата (напр. чрез
   `rsync`/`scp`/git push към хостинга).
 
## Как да съпоставяш продукти, които не са напълно еднакви между магазини
 
Много продукти (особено цимент, гипсокартон, бои) съществуват в различни
марки/класове при различните вериги — няма гарантирано 1:1 съответствие.
Препоръчан подход:
 
1. **Съпоставяй по спецификация, не по бранд.** За цимент това означава:
   клас якост (напр. 42.5R/42.5N), тегло на чувала (25 кг), тип (портланд/
   бял и т.н.). За плочки: размер + тип покритие. За боя: обем + тип
   (латекс/фасадна/силиконова).
2. **Ако точен клас липсва при даден магазин**, вземи най-близкия наличен
   еквивалент и **задължително добави `matchNote`** на нивото на продукта
   в `data/products.json`, обясняващ разликата — така потребителят вижда
   предупреждението директно на страницата за сравнение. Пример (реален):
```json
   "matchNote": "При Praktiker е клас 42.5R (HOLCIM), а при HomeMax наличният
   еквивалент е клас 42.5N (TITAN) — близки, но не абсолютно същите
   характеристики."
```
 
3. **Ползвай полето `label` във всяка оферта**, за да покажеш точното
   заглавие на продукта в конкретния магазин (различно е от общото
   `name` на продукта) — вижда се на страницата за детайлно сравнение
   (`product.html`), за да е ясно на потребителя какво точно сравнява.
4. **Ако даден магазин просто няма разумен еквивалент**, не го добавяй
   към офертите на този продукт — по-добре по-малко, но коректни
   съпоставки, отколкото подвеждащо сравнение на различни неща.
5. Домейните на магазините: `praktiker.bg`, `home-max.bg` (не
   `homemax.bg`!), `inex-bg.com` (не `inex.bg`!, и е основно за
   инструменти — не всички строителни материали ще имат съответствие
   там), `gstroy.bg`, `domestico.bg`, `bauhaus.bg`.
6. `gstroy.bg` рендира цената на продукта през JavaScript — статичният
   HTML съдържа описанието, снимките и meta таговете, но НЕ цената.
   Затова `scrape_gstroy.py` използва Playwright (headless Chromium)
   вместо обикновен `requests.get()`. Виж докстринга в началото на файла
   за подробности, и пусни `playwright install chromium` след
   `pip install -r requirements.txt`.
## Формат на изходните данни
 
`run_all.py` трябва да произведе точно същия формат като
`data/products.json` в главната папка — виж го за пример. Най-важното:
 
```json
{
  "lastUpdated": "2026-07-02T09:00:00+03:00",
  "stores": [ { "id": "praktiker", "name": "Praktiker", "url": "..." }, ... ],
  "categories": [ "Цимент и бетон", ... ],
  "products": [
    {
      "id": "cement-42-5-25kg",
      "name": "Цимент 42.5, 25 кг",
      "category": "Цимент и бетон",
      "unit": "чувал",
      "matchNote": "По избор — обясни разлика в спецификацията между магазините.",
      "offers": {
        "praktiker": {
          "price": 10.74,
          "image": "https://.../snimka.jpg",
          "url": "https://praktiker.bg/.../p/496669",
          "label": "ЦИМЕНТ HOLCIM CEM II/B-LL 42.5R, 25 кг"
        }
      }
    }
  ]
}
```
 
Когато си готов с реален скрапер за поне 1-2 магазина, прати ми
резултата (или линк към страница на продукт) и ще ти помогна да
доизчистим селекторите и да добавим останалите магазини.
