# Настройка на акаунти + изпращане на имейли (Supabase + Resend)

Системата за регистрация/вход и изпращане на имейли използва две безплатни
външни услуги, защото истинска автентикация (пароли, сесии) и изпращане на
имейли изискват истински бекенд — не мога да ти ги "включа" директно в
статичен сайт без тях. И двете имат безплатен план, достатъчен за твоите
нужди в момента.

## Защо точно тези две услуги

- **Supabase** — управлява регистрацията/входа (имейл + парола) и пази
  потребителите в истинска база данни. Безопасно е, защото паролите
  никога не минават през твоя код — Supabase ги хешира и пази вместо теб.
- **Resend** — праща имейла до archipari888@gmail.com, когато някой
  попълни контактната форма.

---

## Стъпка 1 — Създай Supabase проект

1. Отиди на **supabase.com** → Sign up (можеш през GitHub акаунта си).
2. **New Project** → избери организация → дай име (напр. `archipari`) →
   избери парола за базата данни (запази я някъде, не е нужна за сайта,
   но пази я) → избери регион най-близък до България (напр. Frankfurt).
3. Изчакай ~2 минути, докато проектът се създаде.
4. Отиди в **Project Settings** (иконка зъбно колело долу вляво) →
   **API**.
5. Копирай:
   - **Project URL** (изглежда като `https://abcdefgh.supabase.co`)
   - **anon public** ключа (дълъг низ от букви/цифри — това е
     ПУБЛИЧНИЯТ ключ, нормално е да стои във фронтенд кода)

## Стъпка 2 — Попълни ключовете в `js/auth.js`

Отвори `js/auth.js` и замени:
```js
const SUPABASE_URL = "https://YOUR-PROJECT-REF.supabase.co"; // TODO
const SUPABASE_ANON_KEY = "YOUR-PUBLIC-ANON-KEY"; // TODO
```
с реалните стойности от Стъпка 1.

## Стъпка 3 — (По избор) Изключи задължителното потвърждение по имейл

По подразбиране Supabase изисква потребителят да кликне линк в имейла си
преди да може да влезе. Ако искаш регистрацията да работи веднага без
чакане на имейл (по-просто, но по-малко сигурно):

1. Supabase Dashboard → **Authentication** → **Providers** → **Email**.
2. Изключи **"Confirm email"**.

Ако го оставиш включено (препоръчително за истински продукт), Supabase
автоматично праща имейл за потвърждение — не е нужно да пипаш нищо
допълнително в кода, `signup.js` вече показва подходящо съобщение и в
двата случая.

## Стъпка 4 — Създай Resend акаунт

1. Отиди на **resend.com** → Sign up с имейл **archipari888@gmail.com**
   (важно — виж бележката по-долу защо точно този имейл).
2. Потвърди имейла си (Resend ще ти прати линк).
3. Отиди в **API Keys** → **Create API Key** → дай му име (напр.
   `archipari-contact-form`) → копирай ключа (показва се само веднъж!).

**Защо трябва да регистрираш Resend точно с archipari888@gmail.com:**
без верифициран собствен домейн, Resend позволява да пращаш само ДО
имейла, с който е регистриран акаунтът (т.нар. "sandbox" режим). Тъй
като контактната форма и без друго праща само до archipari888@gmail.com,
това е точно каквото ти трябва — без да е нужно да купуваш/верифицираш
домейн за имейли отделно. Ако по-късно поискаш имейлите да идват от
собствен домейн (напр. `contact@archipari.bg`), ще трябва да верифицираш
домейна в Resend (Settings → Domains) — кажи ми тогава и ще актуализирам
`api/contact.js`.

## Стъпка 5 — Добави Environment Variables във Vercel

**Това е задължителна стъпка** — без нея `/api/contact` ще връща грешка.

1. Vercel Dashboard → проекта **ArchiPari** → **Settings** →
   **Environment Variables**.
2. Добави следните три:

| Key | Value |
|---|---|
| `SUPABASE_URL` | същото като в `js/auth.js` (Project URL) |
| `SUPABASE_ANON_KEY` | същото като в `js/auth.js` (anon public ключ) |
| `RESEND_API_KEY` | ключът от Стъпка 4 (тайният, НЕ го слагай никъде във фронтенд кода!) |

3. За всяка промяна избери и трите environment-а (**Production**,
   **Preview**, **Development**), освен ако не знаеш, че ти трябва само
   Production.
4. Натисни **Save**.

## Стъпка 6 — Redeploy

Environment Variables не влизат в сила за вече направени деплойменти.
След като ги добавиш:

```powershell
git add .
git commit -m "Dobavena avtentikatsia i kontaktna forma s email"
git push
```

Vercel автоматично ще редеплойне с новите environment variables.

## Стъпка 7 — Тествай

1. Отвори сайта → **Регистрация** → направи си тестов акаунт.
2. Ако си оставил "Confirm email" включено — провери пощата и кликни
   линка за потвърждение, после влез.
3. Отиди на **Контакти** → трябва да виждаш формата (не заключения
   екран) → попълни и изпрати.
4. Провери пощата на **archipari888@gmail.com** — трябва да пристигне
   имейл с темата и съобщението, а "Отговори" да те насочва директно
   към имейла на потребителя.

## Важно — поверителност на данните

Вече пазиш реални имейли и (хеширани) пароли на потребители. Преди да
пуснеш сайта пред истински хора в по-голям мащаб, добави кратка
**Политика за поверителност**, обясняваща какви данни събираш и защо
(изисква се по ЗЗЛД/GDPR). Кажи ми, ако искаш помощ с това — мога да
направя основен вариант.
